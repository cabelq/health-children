☁️ Servidor SaludInfantil (cloud)

Sincronizá la base cifrada con el servidor dedicado (Node.js + Express + SQLite). El server es opaco: nunca descifra tus datos — solo guarda los bytes cifrados. Soporta múltiples dispositivos con conflict resolution (etag).

Sin configurar.

ℹ️ ¿Cómo funciona?
  1. Configurás la URL del backend y te logueás con tu usuario.
  2. El server te da un JWT (token temporal, 15 min) que se renueva solo.
  3. Cuando hacés cambios, se sube el blob cifrado (con tu AES-GCM key, no la password).
  4. El server lo guarda con un etag (sha256) — si otro device cambió algo, te avisa y evita pisarte.
  5. La password nunca sale del browser. El server no puede descifrar aunque quisiera.
  6. El server usa bcrypt para tu password de login (distinto del salt del cifrado de DB — defensa en profundidad).
